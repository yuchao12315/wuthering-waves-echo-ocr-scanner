## 快速开始

```
整体流程：启动游戏 → 运行OCR扫描 → 导出JSON → Web端导入 → 查看评分 → 计算最优搭配
```

**最短路径（3步上手）**：

1. 安装Python依赖 → 运行OCR扫描 → 得到 `echoes_xxx.json`
2. 启动Web计算器 → 导入JSON
3. 选角色 → 点击「搭配计算」→ 得到最优声骸组合

## 一、OCR扫描工具（Python桌面程序）

### 环境准备

**系统要求**：
- Windows 10 / 11（64位）
- Python 3.9 或更高版本
- 约500MB磁盘空间（含OCR模型）

**安装步骤**：

```powershell
# 进入扫描工具目录
cd scanner

# 方式一：双击一键安装脚本（推荐）
install.bat

# 方式二：手动安装（如果一键脚本失败）
pip install --user -r requirements.txt
```

> **常见安装问题**：如果出现 `[WinError 2] 系统找不到指定文件` 错误，请尝试：
> 1. **以管理员身份运行**：右键 `install.bat` → 以管理员身份运行
> 2. **使用 `--user` 参数**：`pip install --user -r requirements.txt`
> 3. **启用Windows长路径**：以管理员运行PowerShell执行：
>    ```powershell
>    New-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem" -Name "LongPathsEnabled" -Value 1 -PropertyType DWORD -Force
>    ```
>    然后重启电脑再安装
>
> 首次运行时PaddleOCR会自动下载识别模型（~100MB），请保持网络连接。下载完成后即可离线使用。

**游戏设置要求**：

| 设置项 | 要求 |
|--------|------|
| 显示模式 | 全屏 或 无边框全屏 |
| 分辨率 | 1920×1080（推荐）/ 2560×1440 / 1680×1050 |
| 游戏语言 | 简体中文 |
| 界面缩放 | 系统DPI缩放设为100%（推荐） |

### 使用流程

#### 第一步：启动游戏并进入声骸仓库

1. 启动鸣潮，登录角色
2. 打开背包 → 声骸页签（即声骸仓库界面）
3. 将列表滚动到**最顶部**

#### 第二步：运行扫描工具

```powershell
python -m scanner
```

工具启动后自动执行：

```
══════════════════════════════════════════════════
  鸣潮声骸OCR扫描工具 v1.0
══════════════════════════════════════════════════

✓ 检测到鸣潮进程: Client-Win64-Shipping.exe (PID: 12345)
✓ 检测到鸣潮窗口: 1920x1080 at (0,0)

──────────────────────────────────────────────────
请执行以下操作：
  1. 切换到鸣潮游戏窗口
  2. 打开 声骸仓库 界面
  3. 滚动到列表最顶部
──────────────────────────────────────────────────

准备好后按 Enter 开始扫描...
```

按 Enter 后有3秒倒计时，请在此期间切换到游戏窗口。

#### 第三步：等待自动扫描

扫描全程自动执行，分两阶段：

**阶段一 — 列表快扫**

工具自动翻阅所有声骸列表页，通过OCR识别每个声骸卡片上的等级数字，筛选出Lv.25满级声骸的位置。

```
═══ 阶段一：列表快扫（识别满级声骸）═══
  已扫描 42 页，发现 187 个满级声骸
✓ 发现 187 个满级声骸
```

**阶段二 — 详情精扫**

工具逐个点击满级声骸进入详情页，完整识别：
- 怪物名称
- Cost值
- 主词条（类型+数值）
- 副属性（类型+数值）
- 所有副词条（类型+数值）
- 套装名称
- 调谐等级

```
═══ 阶段二：详情精扫（187个声骸）═══
  扫描进度: 187/187

✓ 阶段二完成，共识别 187 个声骸
```

#### 第四步：获取导出文件

```
✓ 已导出 187 个声骸到: echoes_20260618_193042.json

导出完成！请在Web端导入文件: echoes_20260618_193042.json
```

文件保存在当前运行目录下。

### 扫描耗时参考

基于1920×1080分辨率，每页显示6个声骸：

| 仓库规模 | 总声骸数 | 满级声骸(估) | 阶段一 | 阶段二 | 总计 |
|----------|---------|-------------|--------|--------|------|
| 轻度玩家 | 100个 | ~15个 | ~20秒 | ~25秒 | **~1分钟** |
| 中度玩家 | 500个 | ~80个 | ~1分钟 | ~2分钟 | **~3分钟** |
| 重度玩家 | 1000个 | ~200个 | ~2.5分钟 | ~5分钟 | **~7分钟** |
| 极限情况 | 2000个 | ~400个 | ~4.5分钟 | ~9分钟 | **~14分钟** |

> 2560×1440分辨率每页约显示8个声骸，总耗时再减约20%。

### 中断与恢复

扫描期间如果被打断，数据不会丢失：

| 中断情况 | 工具行为 |
|---------|---------|
| 切出游戏窗口 | 自动暂停，提示切回后按Enter继续 |
| 工具意外关闭/崩溃 | 进度已实时保存到 `scan_progress.json` |
| 下次重新运行 | 自动检测未完成记录，提示是否继续 |

恢复对话示例：
```
发现未完成的扫描记录 (已完成 45/187)
是否继续上次扫描? [Y/n]: y
继续扫描剩余 142 个声骸...
```

### 数据校验说明

扫描完成后自动执行以下校验：

| 校验项 | 规则 | 未通过处理 |
|--------|------|-----------|
| Cost合法性 | 必须为1/3/4 | 标记警告 |
| 主词条类型 | 必须与Cost匹配 | 标记警告 |
| 数值范围 | 在合法范围内（含20%容忍度） | 标记警告 |
| 副词条数量 | 调谐等级N → N+1条副词条 | 标记警告 |

存在警告的声骸**仍会导出**，建议在Web端手动检查修正。

### 坐标校准

如果OCR识别率低或点击位置偏移，需要校准坐标配置：

1. 在游戏中截取声骸仓库界面的全屏截图
2. 用画图工具测量各UI元素的像素坐标
3. 编辑 `scanner/config.py` 中对应分辨率的坐标值

关键区域说明：
```python
'list_region'      # 列表页可见声骸区域的 (x, y, width, height)
'card_start_x/y'   # 第一个卡片左上角坐标
'card_width/height' # 单个卡片尺寸
'level_offset'     # 等级数字相对卡片左上角的偏移
'detail'           # 详情页各字段的绝对坐标区域
```

## 二、Web声骸计算器

### 启动方式

Web计算器是纯静态前端应用，有三种运行方式：

**方式一：Node.js（推荐）**

```powershell
# 首次需要构建
npm install
npm run build

# 启动本地服务
npx serve dist
```

浏览器打开 http://localhost:3000

**方式二：Python**

```powershell
npm run build
cd dist
python -m http.server 8080
```

浏览器打开 http://localhost:8080

**方式三：开发模式（有热更新）**

```powershell
npm install
npm run dev
```

浏览器打开 http://localhost:5173

> 所有数据存储在浏览器本地（IndexedDB），不上传任何数据到服务器。

### 功能说明

Web计算器有四个主要页面，通过顶部标签切换：

#### 声骸管理

**导入声骸**（推荐方式）：
1. 点击「导入JSON」按钮
2. 选择OCR扫描导出的 `echoes_xxx.json` 文件
3. 声骸数据自动加载到库存

**手动录入**（少量声骸时使用）：
1. 选择Cost值（1/3/4）
2. 选择套装
3. 选择主词条类型，输入数值
4. 填写副属性（Cost3/4有）
5. 添加副词条（最多5条），逐条选择类型并输入数值
6. 输入梦魇声骸名称（如"梦魇·燎照之骑"），自动匹配加成效果
7. 点击「添加声骸」

**梦魇声骸**：
- 名称含"梦魇"或"共鸣回响"时自动匹配固定加成（属性伤害+技能类型加成）
- 部分梦魇声骸有角色限制（如"共鸣回响·梦魇亚当·重锤"仅限露西/丽贝卡），非限定角色装备时加成不生效
- 声骸卡片以紫色标签显示加成效果，有角色限制时显示橙色提示

**库存浏览**：
- 所有声骸以卡片形式展示
- 选中角色后，每个声骸自动显示对该角色的评分
- 支持按评分/套装/Cost筛选排序
- 点击删除按钮移除声骸

**导出备份**：
- 点击「导出JSON」下载全部库存数据
- 换电脑/清浏览器缓存后可重新导入恢复

#### 搭配计算

1. **选择角色**（必选）— 决定评分权重和伤害计算参数
2. **选择套装偏好**（可选）— 决定计算模式
3. **设置阈值筛选**（可选）— 暴击率/共鸣效率最低要求
4. 点击**「开始计算」**
5. 等待计算完成（通常<3秒）
6. 查看**Top-10最优搭配**结果

**阈值筛选**：
- 可设置暴击率≥X%和共鸣效率≥X%的最低要求
- 填两个阈值→两个都满足才保留；填一个→只检查那一个；不填→不过滤
- 满足阈值的组合≥1时，隐藏低于阈值的方案并显示"已过滤 N 套"
- 满足阈值的组合=0时，显示橙色提示"当前没有组合能满足阈值"
- 每个结果卡片显示暴击率和共鸣效率数值，低于阈值时标红

每个搭配结果显示：
- 5件声骸的详细属性
- 各声骸单独评分
- 5件总评分（满分250）
- 评级标签（C/B/A/S/SS/SSS）
- Cost分配模式
- 暴击率和共鸣效率面板值

#### 已保存套装

- 从搭配计算结果中保存的声骸组合
- 支持替换单个声骸（悬停卡片显示"替换"按钮）
- 每个套装显示伤害计算面板：
  - 攻击力（1位小数）、暴击率、暴击伤害、属性增伤、共鸣效率
  - 按技能分类的期望伤害和暴击伤害明细表
  - 武器选择 + 精炼等级（R1-R5）
  - 命座选择（0-6命）
  - 技能类型筛选（只看特定类型技能的伤害）
- 伤害计算包含完整公式：ATK × 倍率 × 增伤 × 加深 × 暴击 × 防御 × 抗性
- 梦魇声骸加成自动计入对应伤害池（含角色限制过滤）
- Console 输出完整计算日志（`[伤害计算]`前缀）

#### 角色数据库

- 展示全部56个已收录角色
- 显示角色元素属性、武器类型
- 点击角色可设为当前活跃角色（影响评分计算）
- 显示该角色的权重偏好摘要（哪些词条权重高）

### 评分体系说明

采用**权重直乘评分**，对齐 XutheringWavesUID 社区标准：

```
单声骸评分 = (主词条×主权重 + 副属性×副权重 + Σ副词条×副权重) / 该Cost最高分 × 50

角色总分 = 5个声骸各自评分之和，满分250
```

**评级对照表**：

| 分数区间 | 等级 | 含义 | 颜色 |
|---------|------|------|------|
| ≥ 210 | SSS | 理论极限 | 金色 |
| ≥ 195 | SS | 高投入毕业 | 紫色 |
| ≥ 175 | S | 良好 | 蓝色 |
| ≥ 150 | A | 基础水平 | 绿色 |
| ≥ 120 | B | 过渡用 | 灰色 |
| < 120 | C | 建议更换 | 红色 |

**单声骸评级**（50分制）：

| 分数 | 等级 |
|------|------|
| ≥ 42 | SSS |
| ≥ 39 | SS |
| ≥ 35 | S |
| ≥ 30 | A |
| ≥ 24 | B |
| < 24 | C |

**权重说明**：
- 每个角色有独立的权重配置（来自 calc.json）
- 同一声骸对不同角色的评分不同
- DPS角色通常暴击/暴击伤害权重最高
- 辅助角色通常共鸣效率/攻击%权重最高

### 搭配计算器三种模式

| 模式 | 触发 | 说明 | 计算时间 |
|------|------|------|---------|
| 5件套优化 | 选1个套装 | 从该套装的满级声骸中找最优5件，Cost总和=12 | <0.5秒 |
| 双合鸣优化 | 选2个套装 | 套装A≥2件 + 套装B≥2件 + 1件散件，Cost=12 | <1秒 |
| 散件模式 | 不选套装 | 全库存参与，纯评分最大化，Cost≤12 | <2秒 |

**使用建议**：
- 有明确套装目标 → 选1个套装（5件套效果最强但要求库存充足）
- 两个套装都有不错库存 → 选2个套装（2+2灵活度更高）
- 不确定用什么套装 → 散件模式（看看纯评分最高能到多少）

## 四、数据格式与导入导出

### JSON文件格式

OCR导出和Web导出的JSON格式统一：

```json
{
  "echoes": [
    {
      "id": "sha256哈希值",
      "monsterId": 0,
      "monsterName": "无常凶鹭",
      "cost": 4,
      "rarity": 5,
      "level": 25,
      "tuneLevel": 5,
      "sonata": "havoc_eclipse",
      "mainStat": { "type": "CRIT_RATE", "value": 22.0 },
      "secondaryStat": { "type": "FLAT_ATK", "value": 150 },
      "substats": [
        { "type": "CRIT_DMG", "value": 18.6 },
        { "type": "ATK_PCT", "value": 9.4 },
        { "type": "FLAT_ATK", "value": 50 },
        { "type": "RESONANCE_LIBERATION_DMG", "value": 8.6 },
        { "type": "ENERGY_REGEN", "value": 10.0 }
      ]
    }
  ],
  "exportedAt": "2026-06-18T19:30:42",
  "version": "1.0",
  "source": "wuwa-echo-scanner"
}
```

### 词条类型枚举

| 枚举值 | 含义 | 出现位置 |
|--------|------|---------|
| `FLAT_ATK` | 攻击（固定值） | 主/副属性/副词条 |
| `ATK_PCT` | 攻击% | 主/副词条 |
| `FLAT_HP` | 生命（固定值） | 主/副属性/副词条 |
| `HP_PCT` | 生命% | 主/副词条 |
| `FLAT_DEF` | 防御（固定值） | 副属性/副词条 |
| `DEF_PCT` | 防御% | 主/副词条 |
| `CRIT_RATE` | 暴击率 | 主(C4)/副词条 |
| `CRIT_DMG` | 暴击伤害 | 主(C4)/副词条 |
| `ENERGY_REGEN` | 共鸣效率 | 主(C3)/副词条 |
| `ELEM_DMG` | 属性伤害加成 | 主(C3) |
| `HEAL_BONUS` | 治疗效果加成 | 主(C4) |
| `NORMAL_ATK_DMG` | 普攻伤害加成 | 副词条 |

## 五、常见问题

### OCR扫描相关

**Q: 提示"未检测到鸣潮游戏进程"**

A: 确认游戏已启动并完成登录。工具检测的进程名为 `Wuthering Waves.exe` 或 `Client-Win64-Shipping.exe`。如果使用特殊启动器，进程名可能不同。

**Q: 扫描过程中工具点错位置**

A: 坐标配置与实际游戏界面不匹配。请参考「坐标校准」章节修改 `config.py`。常见原因：
- 系统DPI缩放不是100%
- 游戏窗口未完全全屏
- 使用了非标准分辨率

**Q: OCR识别错误率高**

A:
- 确保游戏画质设置为中或以上（过低画质文字模糊）
- 确保无其他窗口叠加遮挡
- 关闭Windows通知中心（避免弹窗遮挡）
- PaddleOCR高精度模式已默认启用

**Q: 扫描只识别了一部分声骸就停了**

A: 列表滚动检测判定连续3次截图相同即认为到底。如果游戏卡顿导致误判，可重新运行（已扫描部分会保存）。

**Q: 会不会被封号？**

A: 本工具不修改游戏文件、不注入进程、不截获网络数据包。操作原理等同于玩家手动截图+手动记录。但请自行评估风险，使用后果自负。

### Web计算器相关

**Q: 浏览器打开后空白**

A:
- 确认通过HTTP服务访问（`http://localhost:xxx`），不能直接双击HTML文件打开
- 检查浏览器控制台是否有报错
- 确认 `dist/` 目录下有 `data/characters.json` 文件

**Q: 导入JSON后没有显示声骸**

A:
- 检查JSON文件格式是否正确（必须有 `echoes` 数组字段）
- 打开浏览器控制台查看是否有解析错误
- 尝试导入少量数据排查问题

**Q: 评分和其他工具对不上**

A: 本工具对齐的是 XutheringWavesUID 的权重直乘体系。不同工具使用不同评分算法（档位评分、达成值评分等），数值不可直接比较。同体系内误差应在±1分以内。

**Q: 搭配计算结果不是我期望的**

A:
- 当前为纯评分最大化，不含合鸣加成量化（Phase 2将加入）
- 不含约束条件（如ER下限），某些组合实战不可用
- 同模式内排名正确，跨模式比较需人工判断合鸣价值

**Q: 数据存在哪里？安全吗？**

A: 所有数据存储在浏览器本地IndexedDB中，不上传任何服务器。清除浏览器数据会丢失，请定期「导出JSON」备份。

### 伤害计算公式

```
期望伤害 = ATK × 倍率 × (1+增伤) × (1+加深) × 暴击系数 × 防御倍率 × 抗性倍率

其中:
  ATK        = (角色基础攻击+武器基础攻击) × (1+攻击%) + 固定攻击
  暴击系数   = critDmg (必暴) 或 critRate×critDmg (期望)
  防御倍率   = (100+角色Lv) / ((99+怪物Lv) + (100+角色Lv)×(1-减防-穿透))
  抗性倍率   = 1 - max(0, 敌人抗性-减抗)
  增伤       = 属性伤害 + 技能类型伤害 (同池加和)

精度规则:
  ATK及乘数: 5位小数 (parseFloat(x.toFixed(5)))
  防御倍率:  9位小数 (parseFloat(x.toFixed(9)))
```
