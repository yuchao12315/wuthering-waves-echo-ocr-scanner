# 鸣潮声骸OCR扫描工具

自动扫描鸣潮游戏内声骸仓库，识别满级声骸属性并导出JSON，供Web端声骸计算器导入使用。

---

## 系统要求

- **操作系统**: Windows 10/11
- **Python**: 3.9+
- **游戏设置**: 全屏或无边框全屏，简体中文
- **支持分辨率**: 1920×1080（1080p，已验证可用，推荐）
- **实验性适配**: 2560×1440（2K）、3840×2160（4K）按 1080p 已验证坐标等比派生，目前未实机验证

---

## 安装

```bash
cd scanner
pip install -r requirements.txt
```

首次运行会自动下载PaddleOCR模型文件（约100MB），请保持网络连接。

---

## 使用方法

### 1. 启动游戏

启动鸣潮并登录角色，确保游戏以**全屏或无边框全屏**模式运行。

### 2. 运行扫描工具

```bash
python -m scanner
```

### 3. 自动检测

工具会自动检测鸣潮进程是否运行：

```
══════════════════════════════════════════════════
  鸣潮声骸OCR扫描工具 v1.0
══════════════════════════════════════════════════

✓ 检测到鸣潮进程: Client-Win64-Shipping.exe (PID: 12345)
✓ 检测到鸣潮窗口: 1920x1080 at (0,0)
```

如果未检测到游戏，工具会提示你先启动游戏。

检测到窗口后会打印当前分辨率适配方式：

```
窗口分辨率: 2560x1440
分辨率适配: 使用 1080p 已验证坐标等比派生: 2560x1440 (sx=1.333, sy=1.333)；该分辨率未实机验证，建议先 --debug 小样本校准
```

1080p 保持原始坐标配置，不走缩放。2K/4K 目前只做等比派生，其他分辨率会按 1920×1080 基准自动缩放并提示需要校准，不作为适配范围。除 1080p 外，建议先加 `--debug` 扫少量声骸，确认点击坐标和 OCR 区域正确后再全量扫描。

### 4. 准备扫描

按照提示操作：

```
──────────────────────────────────────────────────
请执行以下操作：
  1. 切换到鸣潮游戏窗口
  2. 打开 声骸仓库 界面
  3. 滚动到列表最顶部
──────────────────────────────────────────────────

准备好后按 Enter 开始扫描...
```

### 5. 自动扫描

扫描分两个阶段自动执行，全程无需手动干预：

**阶段一 — 列表快扫**：快速翻阅所有声骸，识别等级字段，标记Lv.25满级声骸位置。

```
═══ 阶段一：列表快扫（识别满级声骸）═══
  已扫描 15 页，发现 127 个满级声骸
✓ 发现 127 个满级声骸
```

**阶段二 — 详情精扫**：逐个点击满级声骸进入详情页，完整OCR所有属性。

```
═══ 阶段二：详情精扫（127个声骸）═══
  扫描进度: 127/127
✓ 阶段二完成，共识别 127 个声骸
```

### 6. 导出结果

扫描完成后自动导出JSON文件：

```
✓ 已导出 127 个声骸到: echoes_20260618_193042.json

导出完成！请在Web端导入文件: echoes_20260618_193042.json
```

### 7. 导入Web端

打开声骸计算器Web应用 → 声骸管理页面 → 点击「导入JSON」按钮 → 选择导出的JSON文件。

## 中断与恢复

扫描过程中如果被打断（切出游戏、接电话、工具崩溃等），数据不会丢失：

- **窗口失焦**: 工具自动暂停，提示切回游戏后按Enter继续
- **强制关闭**: 进度已实时保存到 `scan_progress.json`
- **下次启动**: 工具检测到未完成的扫描，提示是否继续

```
发现未完成的扫描记录 (已完成 45/127)
是否继续上次扫描? [Y/n]:
```

## 数据校验

扫描完成后，工具自动对每个声骸执行以下校验：

| 校验项 | 说明 |
|--------|------|
| Cost合法性 | 必须为1、3或4 |
| 主词条类型 | 必须与Cost位置匹配（如Cost4不能出共鸣效率） |
| 数值范围 | 所有数值在合法范围内（含20%OCR误差容忍） |
| 副词条数量 | 调谐等级N → 应有N+1条副词条 |

存在校验警告的声骸仍会导出，但会在终端提示数量，建议在Web端人工复核。

## 注意事项

1. **扫描期间不要操作电脑** — 工具通过模拟鼠标点击和滚轮操作游戏界面
2. **不要遮挡游戏窗口** — 截屏依赖游戏窗口完整可见
3. **关闭系统通知** — 弹出的通知可能遮挡游戏画面导致OCR错误
4. **游戏语言必须为简体中文** — OCR模型仅训练了中文界面
5. **首次使用建议用少量声骸测试** — 确认坐标配置正确后再全量扫描

## 常见问题

**Q: 提示"未检测到鸣潮游戏进程"？**

确认游戏已启动且正在运行。工具检测的进程名为 `Wuthering Waves.exe` 或 `Client-Win64-Shipping.exe`。

**Q: OCR识别错误率高？**

- 确保游戏以原生分辨率全屏运行（非窗口化缩放）
- 确保画面清晰、无遮挡
- 高精度模式（默认）已启用，如果仍有问题可能需要校准坐标配置
- 2K/4K 或其他非 1080p 分辨率首次使用必须加 `--debug`，检查 `debug_output/` 里的列表区和详情区截图是否完整

**Q: 如何校准坐标？**

编辑 `config.py` 中的 1920×1080 基准坐标或为目标分辨率添加精确配置。建议截取游戏界面，用图片编辑器测量各UI元素的像素坐标，重点检查 `list_region`、`card_grid_origin`、`card_width/card_height`、`detail_area` 和 `detail_zones`。

**Q: 支持英文/日文游戏界面吗？**

当前版本仅支持简体中文。其他语言需要额外训练OCR模板和修改文本映射表。

**Q: 会不会被封号？**

本工具不修改游戏文件、不注入进程、不截获网络数据。仅通过截屏+OCR读取界面信息，操作方式与玩家手动截图记录相同。但请自行评估风险，使用后果自负。
